# Plan 16.1: Engineering Standards (TS/Lint/Naming)

## Objective
Establish a clean baseline for the codebase by resolving all linting errors and enforcing consistent naming conventions. This addresses REQ-02 and REQ-03.

## Context
- .gsd/SPEC.md
- .gsd/REQUIREMENTS.md
- .gsd/ROADMAP.md
- package.json
- eslint.config.js
- tsconfig.json

## Tasks

<task type="auto">
  <name>Automatic Lint Resolution</name>
  <files>All source files</files>
  <action>
    Run `npm run lint -- --fix` to resolve all automatically fixable errors.
    Review the remaining errors and fix them manually, focusing on:
    - `no-explicit-any` (replace with proper types)
    - `react-hooks/rules-of-hooks`
    - Unused variables and imports
  </action>
  <verify>npm run lint</verify>
  <done>Zero linting errors or warnings.</done>
</task>

<task type="auto">
  <name>TypeScript Baseline</name>
  <files>All source files</files>
  <action>
    Run `npx tsc --noEmit` and resolve all type errors.
    Prioritize fixing errors in:
    - src/shared/hooks
    - src/features/transfers
    - src/shared/lib
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Zero TypeScript compiler errors.</done>
</task>

<task type="auto">
  <name>Naming Convention Audit</name>
  <files>src/features/**/*, src/shared/**/*</files>
  <action>
    Audit the codebase for naming inconsistencies (e.g., camelCase vs snake_case for variables, PascalCase for components).
    Enforce:
    - camelCase for variables and functions.
    - PascalCase for React components and Types/Interfaces.
    - UPPER_SNAKE_CASE for constants.
  </action>
  <verify>Manual code review</verify>
  <done>Consistent naming conventions throughout the codebase.</done>
</task>

## Success Criteria
- [ ] `npm run lint` passes with 0 errors.
- [ ] `npx tsc --noEmit` passes with 0 errors.
- [ ] Codebase follows the defined naming conventions.
