# Plan 1.1: Fix React Structural Errors

## Objective
Fix critical structural React warnings, primarily `react-refresh/only-export-components`, which breaks Fast Refresh during development, and `react-hooks/set-state-in-effect` bottlenecks that cascade renders.

## Context
- .gsd/SPEC.md
- .gsd/ARCHITECTURE.md
- .gsd/phases/1/RESEARCH.md

## Tasks

<task type="auto">
  <name>Separate Context Providers</name>
  <files>
    - src/shared/lib/AuthContext.tsx
    - src/shared/lib/BranchContext.tsx
    - src/shared/lib/WorkspaceContext.tsx
  </files>
  <action>
    - Ensure these files ONLY export React components.
    - If they export custom hooks (like `useAuth`), move those hooks into separate `.ts` files (e.g., `src/shared/hooks/useAuth.ts`) and import them wherever needed. This fixes `react-refresh/only-export-components`.

<task type="auto">
  <name>Remove synchronous setState in useEffect</name>
  <files>
    - src/features/auth/Settings.tsx
    - src/features/dashboard/AdminCommandCenter.tsx
    - src/features/inventory/AdminPricelist.tsx
    - src/features/reports/components/Calendar.tsx
    - src/shared/lib/BranchContext.tsx
  </files>
  <action>
    - Convert `setState` calls directly inside `useEffect` to use proper state initialization, derived state during rendering, or place them in their own correct callback patterns instead of syncing manually. 
    - Exception for data-fetching callbacks: if the function fetched data (e.g. `fetchProfiles()`) which then calls `setState` internally, calling the async fetched callback is fine but it must not be synchronously chained in a way that generates lint errors. If it's a direct wrap to avoid the warning, make sure dependencies are accurate.
  </action>
  <verify>npm run lint</verify>
  <done>No `react-hooks/set-state-in-effect` errors remain across the codebase.</done>
</task>
