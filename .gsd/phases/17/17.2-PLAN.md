# Phase 17.2: Profit Analysis & Performance Centralization

## Objective
Centralize profit calculation logic and product performance analytics into `ReportService.ts`, move data fetching to a dedicated hook, and standardize the `ProfitAnalysis` dashboard.

## Proposed Changes

### [ReportService.ts](file:///c:/Users/PC/Desktop/Antigravity/Projects/Data%20entry%202/src/features/reports/services/reportService.ts)
- Add `calculateProfitMetrics(sales, expenses)`:
    - Calculates Revenue, COGS (Cost of Goods Sold), Gross Profit, Net Profit.
    - COGS = sum of (quantity * cost_price).
    - Gross Profit = Revenue - COGS.
    - Net Profit = Gross Profit - Expenses.
- Add `calculateCategoryProfit(sales)`:
    - Groups sales by Master Category (parsed from product name `Category > Subcat > Product`).
    - Calculates revenue and profit per category.

### [NEW] [useProfitData.ts](file:///c:/Users/PC/Desktop/Antigravity/Projects/Data%20entry%202/src/features/reports/hooks/useProfitData.ts)
- New hook for fetching sales and expenses data.
- Handles date range and branch filtering.
- Delegates computation to `ReportService`.
- Uses React Query with standardized keys and `staleTime: 5m`.

### [ProfitAnalysis.tsx](file:///c:/Users/PC/Desktop/Antigravity/Projects/Data%20entry%202/src/features/dashboard/ProfitAnalysis.tsx)
- Replace inline `fetchProfitData` with `useProfitData` hook.
- Standardize all currency (₱) and number formatting using `formatUtils.ts`.
- Remove redundant type definitions and inline logic.
