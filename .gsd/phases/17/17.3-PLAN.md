# Phase 17.3: Inventory Reporting & Valuation Planning

## Objective
Centralize inventory valuation logic and historical stock "reverse playback" calculations to improve performance and maintainability.

## Proposed Changes

### 1. `ReportService.ts` Enhancements
- **`InventoryStats` Interface**:
  ```typescript
  export interface InventoryStats {
      totalSKUs: number;
      lowStock: number;
      outOfStock: number;
      totalValue: number | null;
      dailyDelta: number;
  }
  ```
- **`calculateInventoryStats(products, movements)`**: Centralize stats currently in `Inventory.tsx`.
- **`calculateHistoricalStock(products, movements, date)`**: Encapsulate the complex logic from `DailyInventorySummary.tsx`.

### 2. New Hook: `useInventoryReport.ts`
- **Purpose**: Fetch all data needed for historical snapshots (sales, purchases, returns, adjustments, transfers).
- **Queries**:
    - `sales`, `purchases`, `returns`, `refunds`, `adjustments` - all with date range filters.
    - `stock_transfers` - special handling for source/destination branch filters.
- **Computed Value**: Returns the `InventorySnap[]` calculated via `ReportService`.

### 3. Dashboard Refactoring
- **`Inventory.tsx`**: Replace inline `stats` and `movements.reduce` with service calls.
- **`DailyInventorySummary.tsx`**: 
    - Remove `fetchHistory` (lines 46-266).
    - Consume data from `useInventoryReport`.
    - Standardize formatting using `formatUtils.ts`.

## Verification Plan

- Verify that `ReportService` accurately restores stock balances by checking against the live database for `Today`.

### Manual
- Verify `Total Value` visibility for owners.
